# E-Commerce SQL Analysis Project

## Project Overview

This project analyzes a simulated **E-Commerce sales database** using SQL.
The goal is to explore customer purchasing behavior, product performance, and revenue trends using analytical SQL queries.

The dataset represents the typical workflow of an e-commerce platform:

Customer → Order → Products Purchased → Payment

Using this relational structure, multiple SQL queries were written to derive insights about **sales performance, customer behavior, and product popularity**.

---

## Database Schema

The database consists of five core tables.

### 1. Customers_E

Stores customer information.

Columns:

* `customer_id`
* `customer_name`
* `city`

---

### 2. Products

Stores product information.

Columns:

* `product_id`
* `product_name`
* `category`
* `price`

---

### 3. Orders_E

Stores order level information.

Columns:

* `order_id`
* `customer_id`
* `order_date`

Relationships:

* `customer_id` references **Customers_E**

---

### 4. Order_Details

Stores the products included in each order.

Columns:

* `order_detail_id`
* `order_id`
* `product_id`
* `quantity`

Relationships:

* `order_id` references **Orders_E**
* `product_id` references **Products**

---

### 5. Payments

Stores payment information for each order.

Columns:

* `payment_id`
* `order_id`
* `payment_method`
* `amount`

Relationships:

* `order_id` references **Orders_E**

---

## SQL Techniques Used

This project demonstrates several important SQL concepts used in real-world analytics.

### Joins

Used to combine data from multiple tables.

Types used:

* INNER JOIN

Examples:

* Linking customers with orders
* Connecting orders with products
* Matching payments with orders

---

### Aggregations

Used to summarize data.

Functions used:

* `SUM()`
* `COUNT()`
* `AVG()`

Examples:

* Revenue by category
* Monthly sales
* Customer spending

---

### Window Functions

Used for advanced analytical queries.

Functions used:

* `ROW_NUMBER()`
* `DENSE_RANK()`
* `SUM() OVER()`

Examples:

* Ranking products by revenue
* Finding top selling products
* Calculating cumulative sales
* Calculating percentage contribution of products

---

### Common Table Expressions (CTE)

Used to simplify complex queries.

Examples:

* Top selling products ranking
* Customers spending above average

---

### Views

Views were created to simplify repeated analysis.

Example:

* **Order summary view** combining customer, product, quantity, and revenue.

---

### Subqueries

Used to filter aggregated results.

Examples:

* Customers spending above average
* Ranking and filtering results

---

## Key Insights

From the analysis we can observe patterns such as:

* Certain product categories generate higher revenue.
* A few products contribute significantly to total sales.
* Some customers purchase multiple times.
* Payment methods vary, but one method often dominates.
* Customers sometimes purchase across multiple product categories.

These insights reflect the type of analysis commonly performed by **e-commerce companies to optimize sales and understand customer behavior**.
